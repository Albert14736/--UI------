# 领袖 2D 图片渲染机制与避坑指南

在文明6中，替换领袖的 3D 模型为 2D 静态图片（前台立绘、加载界面、外交背景）经常会出现不显示、红叹号或黑屏等严重问题。特别是**领袖前台立绘**，非常容易踩坑。

经过无数次测试（以及长达10小时的血泪教训），我们总结出了以下**绝对可行**的配置模式。请以后制作此类 Mod 时，**严格遵守本指南，千万不要自由发挥！**

---

## 1. 加载界面 (Loading Screen)

加载界面（读条进游戏的界面）的配置相对简单，它**直接使用散图 (DDS)**，不需要打包进 BLP。

**配置位置**：领袖的基础 XML 文件中的 `<LoadingInfo>` 表（例如本项目中的 `Honkai_Leader_HoV.xml`）。

**核心代码示例**：
```xml
<LoadingInfo>
    <!-- ForegroundImage 为前方人物，BackgroundImage 为背后背景 -->
    <Row LeaderType="LEADER_HOH_VOID_HERRSCHER" 
         ForegroundImage="Textures/herrscher_of_the_void_foreground.dds" 
         BackgroundImage="Textures/background.dds" 
         PlayDawnOfManAudio="0" 
         LeaderText="LOC_LOADING_INFO_LEADER_HOH_VOID_HERRSCHER"/>
</LoadingInfo>
```

**🚨 致命坑点及图片尺寸建议**：
1. **绝对路径与后缀**：`ForegroundImage` 和 `BackgroundImage` 的值**必须包含文件所在的相对路径（如 `Textures/`）以及完整的 `.dds` 后缀名！** 绝对不能只写文件名。
2. **加载规则**：这些 DDS 文件必须在 `.modinfo` 文件中通过 `<ImportFiles>` 导入（并且要在 `<Files>` 中声明），供前端 (FrontEndActions) 和局内 (InGameActions) 同时读取。
3. **加载界面尺寸警告**：加载界面请使用常规宽屏比例的高清大图（如 1920x1080），**千万不要**在这里使用局外选人界面的 1024x1024 正方形图片！否则会导致画面被严重拉伸变形，看起来巨奇怪！

---

## 2. 外交界面背景 (Diplomacy Background)

与加载界面类似，局内外交界面的**背景图片**也是**直接使用散图 (DDS)**。

**配置位置**：领袖 XML 文件中的 `<DiplomacyInfo>` 表。

**核心代码示例**：
```xml
<DiplomacyInfo>
    <Row Type="LEADER_HOH_VOID_HERRSCHER" 
         BackgroundImage="Textures/background.dds"/>
</DiplomacyInfo>
```

**🚨 致命坑点及图片尺寸建议**：
* **路径与后缀**：同上，必须带上 `Textures/` 等文件夹前缀和 `.dds` 后缀。必须通过 `<ImportFiles>` 导入。
* **尺寸建议**：局内的外交界面背景图推荐直接使用 **1920x1080** 的高清分辨率，以完美适配游戏内的宽屏显示。

---

## 3. 领袖前台立绘 (Leader Foreground / Fallback) —— 最核心的难点！

这部分决定了玩家在**选人界面的大图**以及**局内交涉时前方出现的领袖图像**。
**注意：这里绝不能像上面一样直接生吃散图！必须使用 ArtDef 并通过 BLP 打包！** 如果尝试绕过打包直接用散图，极大概率会导致图片无法渲染或变成红叹号。

成功渲染 2D 前台的【四大核心条件】缺一不可：

### 条件一：XML 必须设置 SceneLayers="4"
在定义领袖的 XML 文件中（`<Leaders>` 表），必须加上 `InheritFrom="LEADER_DEFAULT"` 和 **`SceneLayers="4"`**。这是告诉游戏引擎：“该领袖没有3D模型，请把 2D 替代贴图渲染在指定的 2D 图层上”。
```xml
<Leaders>
    <Row LeaderType="LEADER_HOH_VOID_HERRSCHER" Name="LOC_LEADER_NAME" InheritFrom="LEADER_DEFAULT" SceneLayers="4"/>
</Leaders>
```

### 条件二：创建 LeaderFallback 的 ArtDef 文件
你需要创建一个特殊的 `.artdef` 文件（本项目中为 `ArtDefs/herrscher_of_the_void_leader.artdef`），其根模板必须为 `LeaderFallback`。这个文件的作用是将领袖 ID 映射到打包好的 BLP 包中的图片。
```xml
<!-- 核心层级展示：Leaders -> DEFAULT -> Animations -> DEFAULT -> BLP Entry -->
<Element class="AssetObjects..BLPEntryValue">
    <!-- m_EntryName 是图片在 BLP 包里的内部名称（不带后缀！） -->
    <m_EntryName text="herrscher_of_the_void_select"/>
    <m_XLPClass text="LeaderFallback"/>
    <m_XLPPath text="Leader.xlp"/>
    <!-- m_BLPPackage 是生成的 BLP 打包文件名 -->
    <m_BLPPackage text="leader_herrscher_of_the_void"/>
    <m_LibraryName text="LeaderFallback"/>
    <m_ParamName text="BLP Entry"/>
</Element>
...
<!-- 最下方集合的 m_Name 必须精准对应你的领袖 ID -->
<m_Name text="LEADER_HOH_VOID_HERRSCHER"/>
```

### 条件三：DEP 文件 (Dependency) 的严格绑定
前台的渲染严重依赖 `.dep` 依赖文件。在你的 `.dep`（如本项目的 `kongzhilvzhe.dep`）中，必须通过它来告诉游戏引擎去读取前面的 ArtDef 和 BLP：

1. **SystemDependencies** 中必须挂载 `LeaderFallback` 消费者，并指向你的 ArtDef：
```xml
<Element>
    <ConsumerName text="LeaderFallback"/>
    <ArtDefDependencyPaths>
        <Element text="herrscher_of_the_void_leader.artdef"/> <!-- 这里关联 -->
    </ArtDefDependencyPaths>
    <LibraryDependencies>
        <Element text="LeaderFallback"/>
    </LibraryDependencies>
    <LoadsLibraries>true</LoadsLibraries>
</Element>
```
2. **LibraryDependencies** 中必须声明你的 BLP 包：
```xml
<Element>
    <LibraryName text="LeaderFallback"/>
    <PackageDependencies>
        <Element text="leader_herrscher_of_the_void.blp"/> <!-- 这里关联 -->
    </PackageDependencies>
</Element>
```

### 条件四：正确的 Modinfo 加载逻辑
1. 你的前台图片必须通过官方 ModBuddy 或 Asset Editor 被编译打包成 `.blp` 文件（位于 `Platforms/Windows/BLPs/` 目录下）。
2. 在 `.modinfo` 文件中，**必须同时在 `FrontEndActions` 和 `InGameActions` 中通过 `<UpdateArt>` 执行你的 `.dep` 文件**。只有这样，选人界面（前端）和外交界面（局内）才能正常触发 ArtDef 读取！
```xml
<UpdateArt id="HONKAI_ART_FRONTEND">
    <File>kongzhilvzhe.dep</File>
</UpdateArt>
```
3. 在底部的 `<Files>` 列表里，一定要确保包含了你的 `.blp`、`.artdef` 和 `.dep` 文件。

### 💡 终极总结（心法）
* **加载图与背景图** 走 **XML直接配路径 + ImportFiles 导入生吃**。
* **领袖前台立绘** 走 **BLP打包 -> ArtDef指定条目 -> DEP文件串联绑定 -> Modinfo用UpdateArt挂载DEP文件** 的四步闭环。其中 XML 必须带有 `SceneLayers="4"` 的保底设置。

无论后续添加多少个领袖，只要严格复制这套机制进行名称替换，就绝不会再出现渲染失败的问题！配路径 + ImportFiles 导入生吃**。
