# 调用原版模型注意事项

如果你不想自己做3D模型，想在你的Mod里直接调用文明6原版的建筑、单位或区域模型，请注意以下几点。

## 1. 战略视图 (Strategic View) 的快速“白嫖”
在 `Buildings.artdef` 或 `Districts.artdef` 中，你可以直接引用原版的战略视图定义。
*   **例子**：本项目中的 `BUILDING_HOH_ST_FREYA` 直接引用了 `Oracle` (神谕所) 的战略视图。
*   在 `StrategicView` 集合下，将 `XrefName` 指向原版的名称（如 `Oracle`, `Oracle_Pillaged`, `Oracle_UnderConstruction`）。

## 2. 3D模型的直接引用 (ArtDefs)
如果你想让你的建筑在地图上看起来像原版的某个建筑：
1.  在 `Buildings.artdef` 的 `Landmarks` 集合里。
2.  添加一个元素，在 `XrefName` 中填入原版模型的名称（通常是 `BUILDING_XXX`）。
3.  **注意**：你不需要在你的 Mod 文件夹里放任何 `.ast` 或 `.mtl` 文件，游戏引擎会自动从根目录里找。

## 3. 图标的别名映射 (最推荐)
如果你想让你的东西用原版的图标，不要复制图片，直接在 `Icons.xml` 里用 `IconAliases`：
```xml
<IconAliases>
    <!-- 我的建筑名 -->          <!-- 原版建筑图标名 -->
    <Row Name="ICON_BUILDING_MY_XYZ" OtherName="ICON_BUILDING_ORACLE"/>
</IconAliases>
```

## 4. 关键点：Dep文件 (Dependency)
*   文明6的 ArtDef 具有继承性。如果你的 Mod 报错找不到模型，请检查你的 `.dep` 文件。
*   确保你的 `.dep` 包含了对原版（BaseGame）或特定 DLC（如 Expansion2）的引用。通常 Asset Editor 生成的 `.dep` 会自动处理这个。

## 5. 原版路径参考
*   建筑模型通常在：`BaseGame/ArtDefs/Buildings.artdef`
*   区域模型通常在：`BaseGame/ArtDefs/Districts.artdef`
*   如果你想“借鉴”某个 DLC 的模型（如 Expansion 2 的运河），你的 Modinfo 必须声明对该 DLC 的依赖，否则玩家如果不买那个 DLC，你的模型就会变成大红叉。
